# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run build              # Full build: ships + backgrounds + HTML
npm run build:svg          # Copy SVGs only (no rasterization)
npm run build:sprites      # Rasterize + HTML only
node scripts/build.js --ships-only       # Ships section only
node scripts/build.js --backgrounds-only # Backgrounds section only
npm run preview            # Serve docs/ on http://localhost:3000
```

No test suite. Output goes to `docs/` (GitHub Pages source).

## Architecture

### Project purpose

Procedural pixel-art asset generator for damoigmai (a Phaser 4 side-scrolling shoot'em up). Assets are published to GitHub Pages and loaded by the game at runtime from the CDN URL `https://rpigab.github.io/pixelagen/`.

### Three sections (ongoing migration from all-procedural to CDN assets in damoigmai)

| Section | Source | Output | Status |
|---|---|---|---|
| **Ships** | `src/ships/*.svg` | `docs/sprites/<variant>/<size>/<name>.png` | complete — 19 ships |
| **Backgrounds** | `scripts/generate-backgrounds.js` | `docs/backgrounds/space/*.png` | ESPACE done (4 layers) |
| **Enemies** | `src/enemies/` | `docs/enemies/` | placeholder |

### Build pipeline

```
scripts/build.js          orchestrator — calls the two pipelines below + generates all HTML
scripts/rasterize.js      SVG file → PNG variants (sharp + quantize)
scripts/quantize.js       uniform RGB cube palette + optional Bayer 4×4 dithering
scripts/generate-backgrounds.js  procedural SVG strings → 160×90 PNG layers (sharp)
```

### Ship rasterization

- SVG viewBox: **256×128** (2:1 aspect ratio)
- Output sizes: **32×16**, **64×32**, **128×64** (always half-height)
- 6 color variants: `full`, `256colors`, `256-dithered`, `64colors`, `16colors`, `16-dithered`
- CDN backward-compat: SVGs are also copied to `docs/svgs/` and PNGs stay at `docs/sprites/` — never move these paths.

### Background layers

Art resolution: **160×90 pixels** — matches `PIXEL_SCALE=3` in damoigmai (480÷3 × 270÷3).  
Each PNG tiles horizontally via Phaser `tileSprite`. Layers are loaded in `BootScene.preload()`.

ESPACE layers (scroll speed is relative; layer 0 = slowest):

| Key | Opacity | Role in game |
|---|---|---|
| `space_layer0` | Opaque | Replaces Phaser solid bg color |
| `space_layer1` | Transparent | Nebula color wash |
| `space_layer2` | Transparent | Mid-density stars |
| `space_layer3` | Transparent, sparse | Foreground — renders **above** enemies; keep elements in top/bottom strips |

Backgrounds are generated programmatically (seeded LCG random) in `generate-backgrounds.js` — no SVG source files. To add a new world, add a new section of `svgLayerN()` functions and a new `LAYERS` entry.

### Static site (`docs/`)

```
docs/
  index.html          landing page (3 section cards)
  ships/              ship galleries (index, svgs, sprites)
  backgrounds/        parallax preview with speed/pause/toggle controls
  enemies/            placeholder
  svgs/               SVG copies (CDN backward-compat)
  sprites/            PNG output (CDN backward-compat — do not move)
```

HTML is fully generated by `build.js` — do not hand-edit files in `docs/`.
